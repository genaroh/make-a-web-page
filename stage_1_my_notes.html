BASIC CONCEPTS OF THE WORLD WIDE WEB (WWW)

How the WWW works

The World Wide Web (WWW) is a collection of documents in the HyperText Markup Language (HTML) format. Hyperlinks are what make the web web-like. The WWW uses special computers called servers. Servers are computers that are optimized for sitting in a closet and hosting files. A browser makes requests to servers via the Internet using HTTP. HTTP is a protocol for communicating with servers.

The 3 Programming Languages of Web Development

The 3 programming languages of web development are HTML, CSS, and JavaScript. As previously mentioned, HTML controls the structure of a web page. CSS stands for Cascading Style Sheets and is used to control the style (i.e., look and formatting) of a web page written in a markup language. JavaScript provides interactive components to a web page.

A CLOSER LOOK AT HTML

How HTML works

HTML is the standard markup language for creating web pages. HTML controls the structure of a web page and consists of:

  • Text content (what you see)
  • Markup (what the content looks like)
  • References to other documents (e.g. images, video)
  • Links to other pages

HTML uses tags to markup a page. There are opening and closing tags. An element consists of opening tag, its content, and it's closing tag. An element has form '<tag_name tag_attribute="some_val"> content_to_display </tag_name>'. A tag does not always have an attribute. Furthermore, some elements do not have a closing tag. For instance, the '<img src="some_val" alt="some_other_val">' A list of HTML tags in alphabetical order can be found on the W3SCHOOLS website.

The general structure of an HTML document

The general structure of an HTML document is shown below. An HTML document contains a declaration at the top that specifies the file is of type HTML, an html element, a head and body section inside of the html element.

  • <!DOCTYPE HTML>
  • <html>
    • <head>
      • <title>some_title</title>
    • </head>
    • <body>
      • Web page body
    • </body>
  • </html>

An online HTML validator can be used to validate HTML code.

Everything on a web page is a box

It's amazing that everything on a web page is a box. Web developers will boxify a design. Boxify is a technical term. To boxify a design means to take a web page mock-up and to determine the boxes needed for every piece of information in the mock-up (e.g. an image or text). Boxes can exist inside of boxes inside of boxes, and so on, until one gets to a very simple element such as an image or a paragraph. Boxes can be created on a web page by using the <div> tag.

Document Object Model

The Document Object Model (DOM) is a cross-platform and language-independent standard for representing and interacting with objects in HTML (and other markup languages). In this standard, HTML elements (i.e., an opening and closing tag with their content) form nodes. These nodes are further organized into a tree structure known as the DOM tree. Every web pages is represented as a DOM tree.

THE SIGNIFICANCE OF CSS

How CSS works

CSS is used to add style to a web page. Style refers to the look or appearance of a web page. CSS essentially selects an HTML tag or class to style. Hence, CSS uses selector. In either case, a rule is specified to add the desired style. The rule employs a 'attribute: value;' pair. For example,

  • p {
  • color: red
  • }

makes it so every p tag uses red font color. Similarly,

  • .description {
  • background-color: blue
  • }

gives a blue background color to every instance of the description class. More information about CSS selectors can be found on the CSS-TRICKS website.

There are 3 ways to include CSS styling in a web page:

  1. Write CSS in the head of the HTML. This is good for small projects.
  2. Write the style inline with the HTML. This is generally a very bad idea because it leads to a lot of repeated code and can lead to a lot of errors.
  3. Link the HTML to a separate CSS file. This is good for large projects. For example, for a main.html and main.css file, a link tag would be added inside of the head element:
    • <head>
      • <link rel="stylesheet" href="main.css">
    • </head>
    The main.html and main.css should be in the same folder.

The CSS reference has more information about CSS. An online CSS validator can be used to validate CSS code.

Avoiding code repetition

Inheritance is a key feature of CSS that helps avoid code repetition. That is, it helps a programmer from having to repeat the same code over and over again. Inheritance means descendant elements inherit CSS properties of an ancestor element that encloses them. This is best illustrated with an example. Suppose the following is in a CSS file

  • h1 {
  • color: pink;
  • }
and this is in a HTML file
  • <h1>
    • This is to <em> illustrate </em> the concept of inheritance.
  • </h1>

Then unless a color has been assigned to the em element, the emphasized element will print text in pink color. This behavior arises from the em tag inheriting the color style of the parent h1 element. Inheritance prevents certain styles from being declared over and over again because those styles can be specified in a style sheet (see the 3rd way to include CSS styling in a web page).
